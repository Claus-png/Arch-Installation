# Arch-Installation

## Что это такое

Это продвинутый TUI-установщик Arch Linux с пошаговым мастером, поддержкой резервного копирования и встроенной подготовкой к первому запуску. Основной запуск выполняется через `install/install_tui.sh`.

## Архитектура проекта

- `install/install_tui.sh` — точка входа, запуск мастера и маршрутизация режимов.
- `install/lib/` — ядро установщика: UI, состояние, диски, система, пакеты.
- `install/stages/` — отдельные шаги мастера: приветствие, preflight, диск, пользователь, пакеты, итог, установка.
- `install/post/oobe.sh` — первый запуск после установки.
- `install/backup_tui.sh` — отдельный мастер восстановления и резервного копирования.
- `config/defaults.conf` — значения по умолчанию для пользователя, хоста и паролей.

## Как запускать

### Стандартный запуск мастера

```bash
bash install/install_tui.sh
```

### Dry-run

Проверка сценария без реальных изменений:

```bash
bash install/install_tui.sh --dry-run
```

### Открыть OOBE после установки

```bash
bash install/install_tui.sh --oobe
```

## Что делает установщик

1. Проверяет окружение: UEFI, интернет, время, память, доступность `whiptail`.
2. Определяет CPU/GPU и подготавливает параметры для драйверов и zram.
3. Позволяет выбрать диск и включить LUKS.
4. Запрашивает имя пользователя, хост и пароли.
5. Позволяет выбрать пакеты.
6. Выполняет `pacstrap`, настройку загрузчика, локали, пользователей и конфигурации.
7. Завершает установку и предлагает перезагрузку.

## Резервное копирование и восстановление

Резервное копирование вынесено в отдельный TUI-модуль `backup_tui.sh`.

### Основные сценарии

- `bash install/backup_tui.sh` — обычный запуск из установленной системы.
- `bash install/backup_tui.sh --live-usb` — запуск с Live USB / archiso.
- `bash install/backup_tui.sh --chroot-mode` — внутренний режим для chroot, не запускать вручную.

### Что сохраняется

- домашние конфиги пользователя;
- темы, шрифты и настройки KDE;
- SSH/GPG-бэкапы;
- список пакетов и другие подготовленные данные.

## Конфигурация по умолчанию

Значения можно переопределить через `config/defaults.conf` или переменные окружения перед запуском:

- `DEFAULT_USERNAME`
- `DEFAULT_HOSTNAME`
- `DEFAULT_USE_LUKS`
- `DEFAULT_USER_PASSWORD`
- `DEFAULT_ROOT_PASSWORD`

## Рекомендации

- Используйте `--dry-run`, чтобы проверить поток без изменений.
- Перед установкой обязательно убедитесь, что выбран правильный диск.
- После установки перезагрузите систему для входа в новую конфигурацию.

## Поддержка и логирование

Логи сохраняются в `/var/log/install.log`.

## Что улучшено в этом релизе

- весь основной код живёт в `install/`;
- мастер разбит на отдельные стадии;
- улучшены сообщения и итоговый обзор;
- добавлены безопасные значения по умолчанию для паролей;
- OOBE и восстановление закреплены как отдельные сценарии.
