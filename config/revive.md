# Code Review — Arch Installer (Teddy Installer)

> Обновлённый статус: закрытыми считаются проблемы, которые уже реализованы в текущей версии кода.

## Закрыто в текущем коде

- #1 `_step` / gauge в live/fresh — исправлено за счёт перенаправления `stdout` в `fd 3` и записи прогресса через `_step`.
- #2 `_try` передаёт description как команду — исправлено.
- #3 `PROGRESS_FD` не инициализируется мусорным FD 3 — сейчас используется пустое значение до открытия канала.
- #4 `finish_progress_bar` без `eval` — использован `exec {PROGRESS_FD}>&-`.
- #5 `_log` не создаёт директорию на каждый вызов — директория создаётся один раз в точке входа.
- #6 `_die` пишет FATAL в лог — исправлено.
- #10 `validate_disk_path` дублируется — оставлен только в `lib/disk.sh`.
- #11 dry-run помечает `run_stage` как done — исправлено.
- #12 `reset_install_state` не сбрасывает при каждом запуске — сброс только по `--reset-state`.
- #13 `configure_mkinitcpio` не даёт `blockencrypt` — вставка `encrypt` безопасна.
- #14 `mkinitcpio` теперь падает фатально при ошибке.
- #20 `DISK_LIST` проверяется на пустой список — есть явная `_die`.
- #27/28/29 пароли и hostname — подтверждение, запрет пустого пароля и `validate_hostname` уже реализованы в `stage_04_user_config`.
- #36/37 pacstrap и bootloader — ucode ставится только по CPU, добавлена `arch-lts.conf`.
- #42 поиск бэкапов в OOBE не использует `find /`.
- #46/47/48/49/50/51/52 post/live/fresh и вспомогательные пути — уже приведены к актуальному состоянию.
- #53 дефолтные пароли пустые.
- #54 `|| true` на критичных командах не используется.
